# User Guide (Local Agent)

Актуальная инструкция под текущий UI `9second_capture`.

## 1. Подготовка

1. Установите зависимости:

```bash
cd <project-directory>
pip install -r requirements.txt
```

2. Для MP3-конвертации нужен `ffmpeg` (если используете локальный запуск без .app):

```bash
brew install ffmpeg
```

3. Для локального LLM/embeddings-провайдера (например, Ollama):

```bash
brew install --cask ollama
open -a Ollama
ollama pull llama3.1:8b
```

## 2. Запуск

```bash
cd <project-directory>
python3 scripts/run_local_agent.py
```

Откройте UI по адресу из логов запуска или из мастера установки.

## 3. Режим записи

В блоке `1. Режим` выберите один из режимов:

- `Браузер: экран + звук`
- `API/файл: загрузка аудио`
- `Ссылка: quick fallback`

## 4. Запись

1. В блоке `2. Запись` включите `Добавлять микрофон в запись` при необходимости.
2. Нажмите `Старт`.
3. В браузерном системном окне выберите вкладку/экран и включите `Share audio`.
4. Нажмите `Стоп` после завершения встречи.

Важно: после `Стоп` сохраняется аудио (MP3). Транскрипт автоматически не запускается.

## 5. Аудио (вкладка Результаты -> Аудио)

1. Выберите запись в списке.
2. Нажмите `Сохранить MP3`, чтобы сохранить файл в вашу папку.
3. Для внешнего файла используйте `Импорт MP3`.

Важно: импорт MP3 только добавляет запись в агент, без автоматической транскрипции.

## 6. Транскрипция (вкладка Результаты -> Транскрипция)

1. Выберите запись для `Raw` или `Clean`.
2. Укажите имя TXT файла.
3. Нажмите `TXT` для нужного варианта.

Это явный ручной запуск STT. Без нажатия кнопки транскрипт не строится.

## 7. LLM (вкладка Результаты -> LLM)

1. Прикрепите TXT/CSV/JSON/MD через `Attach files`.
2. Введите запрос (например: таблица, JSON, CSV, summary).
3. Нажмите `Отправить`.
4. Для очистки истории и результатов используйте `Очистить`.

LLM работает только по ручному запросу пользователя.

## 8. Настройки моделей (блок Обработка)

- `STT модель`: сканирование и переключение Whisper модели.
- `LLM модель`: сканирование и выбор chat-модели.
- `Embeddings модель`: отдельный выбор embedding-модели (не используется как chat-LLM).

## 9. Рекомендованные значения

- Быстрее STT: `small`
- Точнее STT: `medium` или `large-v3` (сильнее нагрузка)
- Для локальной LLM: держите активной установленную chat-модель Ollama (`llama3.1:8b` и т.п.)
