# 9second_capture: Частые проблемы и решения (системный звук / MP3 / отчёты)

Этот файл нужен для быстрых проверок, когда:

- `System` в диагностике показывает `1%` или около того.
- MP3 после записи создаётся долго или не сохраняется.
- Непонятно, почему не формируется TXT-отчёт.
- Имя записи/файла путается в UI.

## 1. Симптомы и что это обычно значит

### Симптом: `System: 1%`, а `Mic` выше

Обычно это значит:

- агент слышит в основном микрофон;
- системный звук (YouTube/встреча) не попадает в `BlackHole`;
- маршрутизация macOS настроена не полностью.

Признак в артефактах:

- в `raw.txt` больше вашего голоса, чем речи из YouTube/встречи.

### Симптом: после `Стоп` MP3 не сохраняется / появляется ошибка MP3

Обычно это значит:

- запись завершилась, но в системе нет `ffmpeg`;
- есть `backup_audio.webm`, но нет конвертации в `meeting_audio.mp3`.

### Симптом: жму `Сформировать TXT`, но ничего не понятно

Проверьте:

- выбрана запись;
- выбрано `Raw` или `Clean`;
- указано имя файла отчёта (поле имени отчёта).

## 2. Быстрая проверка (за 1 минуту)

1. В UI выберите режим `Драйвер: системный звук`.
2. Источник должен быть `BlackHole 2ch`.
3. Включите YouTube-видео с речью.
4. Во время теста молчите 10 секунд.
5. Нажмите `Старт`.
6. Смотрите диагностику:
   - `System` должен пульсировать заметно выше `1%`.
   - `Mic` может быть ниже.

Если `System` стабильно `1%`, переходите к разделу настройки macOS ниже.

## 3. Пошаговая настройка macOS (BlackHole + Multi-Output Device)

Цель:

- звук идёт в уши/колонки (чтобы вы слышали);
- этот же звук одновременно идёт в `BlackHole` (чтобы агент записывал).

### Шаг 1. Открыть Audio MIDI Setup

1. Нажмите `Cmd + Space`.
2. Введите `Audio MIDI Setup` / `Аудио MIDI`.
3. Откройте приложение.

### Шаг 2. Создать Multi-Output Device (если ещё не создано)

1. Нажмите кнопку `+` внизу слева.
2. Выберите `Create Multi-Output Device` (`Многовыходное устройство`).

### Шаг 3. Настроить Multi-Output Device (важно)

В левой колонке выберите `Многовыходное устройство`.

Справа выставьте:

1. `Первичное устройство` = `BlackHole 2ch`
2. `Частота дискретизации` = `48,0 кГц`

В таблице ниже:

1. В колонке `Вкл.` включите оба устройства:
   - `BlackHole 2ch`
   - ваши колонки/наушники (например `Динамики MacBook Air`)
2. В колонке `Коррекция смещения`:
   - для **физического** устройства (`Динамики MacBook Air` / наушники) -> **галочка ВКЛ**
   - для `BlackHole 2ch` -> **галочка ВЫКЛ**

Важно:

- Частота `48,0 кГц` должна быть у `BlackHole`, у физического устройства и у `Многовыходного устройства`.

### Шаг 4. Сделать Multi-Output Device системным выходом

1. Откройте `Системные настройки` -> `Звук`.
2. В разделе `Выход` выберите `Многовыходное устройство`.
3. Не выбирайте `BlackHole 2ch` напрямую в качестве системного выхода.

### Шаг 5. Проверить громкость источника

Перед тестом:

- громкость macOS: `80–100%`;
- громкость YouTube: `100%`;
- вкладка браузера: не `mute`;
- если есть расширение/микшер вкладок, убедитесь, что вкладка не приглушена.

### Шаг 6. Настроить агент

В `9second_capture`:

1. Режим: `Драйвер: системный звук`
2. Источник: `BlackHole 2ch (Virtual)` (или похожее имя с `BlackHole`)
3. Запустите запись (`Старт`)
4. Для теста молчите `10–15 сек`, чтобы не забивать системный звук микрофоном

### Шаг 7. Проверить результат

Признаки правильной настройки:

- `System` не стоит стабильно на `1%`;
- в `raw.txt` есть речь из YouTube/встречи;
- предупреждение про тихий системный звук появляется редко или исчезает.

## 4. Что делать, если `System` всё равно 1%

Проверьте по списку:

1. В `Системные настройки -> Звук -> Выход` выбрано именно `Многовыходное устройство`.
2. В `Audio MIDI Setup` в `Многовыходное устройство` включены оба устройства (`BlackHole` + динамики/наушники).
3. `Первичное устройство` = `BlackHole 2ch`.
4. `Коррекция смещения` включена у физического устройства, а не у `BlackHole`.
5. В агенте выбран `BlackHole`, а не микрофон.
6. YouTube/вкладка браузера не приглушены.
7. Перезапущен браузер после смены системного выхода.
8. Перезапущен агент после перенастройки Audio MIDI.

### Отдельно про AirPods / Bluetooth

Для первого теста лучше использовать:

- встроенные динамики MacBook, или
- проводные наушники.

Bluetooth-устройства могут давать нестабильные уровни/переключать профиль.

## 5. MP3 не создаётся / долго создаётся

### Почему это происходит

После `Стоп` агент:

1. завершает запись;
2. делает `finish` (финальная обработка);
3. собирает/конвертирует аудио в `MP3`;
4. только потом предлагает сохранить файл.

Это нормально, что появляется задержка.

### Если MP3 не отдается (ошибка / 404)

Частая причина: не установлен `ffmpeg`.

Установите:

```bash
brew install ffmpeg
```

После установки:

1. перезапустите агент;
2. повторите запись/сохранение MP3.

## 6. TXT-отчёт не формируется (или непонятно, что делать)

Перед нажатием `Сформировать TXT` проверьте:

1. выбрана запись в блоке `1) MP3 после встречи`;
2. выбран источник `Raw` или `Clean`;
3. указано имя файла отчёта (поле имени отчёта);
4. если есть кнопка `Сохранить TXT`, её используйте после генерации.

Если отчёт строится долго:

- это может быть из-за LLM (например Ollama отвечает медленно или по таймауту);
- в таком случае агент может сделать fallback-отчёт.

## 8. Что приложить при ошибке (чтобы быстро разобраться)

Если что-то опять не работает, приложите:

1. скрин `Audio MIDI Setup` (`Многовыходное устройство`);
2. скрин `Системные настройки -> Звук -> Выход`;
3. скрин блока диагностики в агенте (`System`/`Mic`);
4. `meeting_id` записи;
5. текст ошибки/подсказки из UI.
