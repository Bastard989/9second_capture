## Общая логика работы

В агенте действия запускаются поэтапно и вручную:

1. сначала получается аудио;
2. потом по кнопке строится транскрипт;
3. потом по кнопке строится результат через LLM или запускается RAG-поиск.

Такой подход помогает избежать лишней нагрузки и делает процесс прозрачным.

## Основные режимы

### Браузер: экран и звук

Используется, когда нужно записать вкладку или экран. Для корректной записи звука в системном окне браузера нужно включить `Share audio`.

### API/файл: загрузка аудио

Используется, когда MP3 уже есть и его нужно добавить в агент для дальнейшей ручной обработки.

### Ссылка: quick fallback

Используется для быстрого запуска записи по ссылке встречи.

## Работа с аудио

После записи или импорта аудио пользователь может:

- выбрать запись в списке;
- сохранить MP3 в нужную папку;
- переименовать запись внутри агента.

Важно: импорт MP3 не запускает транскрибацию автоматически.

## Работа с транскриптами

Во вкладке `Транскрипция` можно вручную построить текстовый файл.

Рекомендуемая логика:

- запускать только тот вариант транскрипта, который действительно нужен;
- для скорости использовать более легкую STT-модель;
- для качества использовать более точную модель, если позволяет время.

## Работа с LLM

Во вкладке `LLM` пользователь прикрепляет файлы и задает понятный запрос. На основе файлов LLM может построить:

- таблицу;
- JSON;
- CSV;
- summary;
- произвольный текстовый отчет.

Важно: LLM не должна получать embeddings-модель вместо chat-модели.

## Работа с RAG

RAG в текущей логике работает по готовым TXT-файлам, а не по аудио.

Сценарий такой:

1. прикрепить готовые TXT-транскрипты;
2. задать вопрос или задачу;
3. выбрать `Top-K`;
4. получить ответ и найденные фрагменты, на которых он основан.

`Top-K` — это число фрагментов текста, которые система берет в ответ. Меньшее число обычно дает более компактный ответ, большее — более широкий охват.

## Рекомендуемый сценарий работы

1. Записать встречу или импортировать MP3.
2. Сохранить MP3.
3. Построить TXT-транскрипт вручную.
4. Прикрепить TXT в LLM или RAG.
5. Получить нужный результат.

## Что важно помнить

- транскрипт не строится сам без команды;
- LLM не запускается сама без команды;
- RAG работает по текстам, а не по аудио;
- для длинных встреч лучше заранее понимать, какой именно результат нужен пользователю.
