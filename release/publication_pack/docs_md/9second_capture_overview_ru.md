# 9second_capture: обзор продукта

## Что это такое

`9second_capture` — локальный агент для записи встреч, ручной транскрибации аудио в текст и генерации результатов через LLM.

Продукт помогает пройти понятную цепочку:

1. сохранить аудио встречи;
2. по запросу получить TXT-транскрипт;
3. по запросу построить таблицу, JSON, CSV, summary или другой результат;
4. при необходимости сравнивать готовые TXT-транскрипты через RAG-поиск.

## Для кого подходит

Агент подходит для:

- команд, которые проводят интервью, статусы, планирования и синки;
- сотрудников, которым нужно быстро перейти от записи к тексту и отчету;
- команд, которым важна локальная обработка и управляемый ручной сценарий.

## Как работает агент

В агенте сознательно разделены три отдельные задачи:

- `STT` превращает аудио в текст;
- `LLM` строит результаты по готовым файлам;
- `RAG` ищет и сравнивает уже готовые TXT-файлы по смыслу.

Агент не запускает транскрибацию и генерацию автоматически без команды пользователя. Это снижает лишнюю нагрузку и делает процесс более предсказуемым.

## Основные возможности

- запись браузерной вкладки или экрана со звуком;
- импорт готовых MP3-файлов;
- ручная транскрибация через STT;
- ручная генерация отчетов и структурированных форматов через LLM;
- RAG-поиск по готовым TXT-транскриптам;
- раздельная настройка LLM, embeddings и STT провайдеров.

## Универсальная модель провайдеров

Приложение не привязано к одному стеку. Отдельно настраиваются:

- `LLM provider`;
- `Embeddings provider`;
- `STT provider`.

Поддерживаются локальные и внешние сценарии. Это позволяет использовать локальный стек, облачные API или гибридную схему под требования команды.

## Безопасность и данные

По умолчанию агент ориентирован на локальную работу. Аудио, транскрипты и файлы результатов хранятся локально, если команда не настраивает отдельное внешнее хранилище или внешних провайдеров.

Для публикации и массового внедрения не рекомендуется:

- хранить реальные ключи и секреты в репозитории;
- публиковать локальные пути пользователей;
- публиковать персональные данные в примерах и скриншотах.

## Что получает пользователь на выходе

На выходе команда получает:

- MP3-файл записи;
- TXT-транскрипт;
- отчет через LLM;
- структурированные данные в формате `JSON`, `CSV` или таблицы;
- при необходимости — ответ RAG с опорой на найденные текстовые фрагменты.
