# 9second_capture: пакет для публикации в Яндекс Вики

## Цель пакета

Этот набор материалов подготовлен для размещения в корпоративной Яндекс Вики как понятный стартовый комплект для пользователей и администраторов.

## Что рекомендуется разместить на странице

1. краткое описание продукта;
2. ссылку на ZIP-архивы с бинарниками;
3. инструкцию по установке;
4. краткое руководство пользователя;
5. типовые ошибки и решения;
6. при необходимости — описание архитектуры и провайдеров.

## Как лучше оформить страницу

Рекомендуемая структура страницы:

1. Что это за агент
2. Что умеет
3. Как установить
4. Как запустить
5. Как использовать
6. Типовые ошибки
7. ZIP-файлы и Word-документы

## Что прикладывать файлами

Удобно прикладывать:

- ZIP-архивы бинарников по платформам;
- Word-файлы с инструкциями;
- при необходимости отдельные PDF или PNG-скриншоты.

## Что не стоит публиковать

Не рекомендуется размещать в открытых для широкого круга пользователей материалах:

- реальные API keys и client secrets;
- service account JSON;
- локальные пути конкретного сотрудника;
- личные данные из примеров встреч и транскриптов;
- внутренние runbook-документы для администрирования, если они не нужны конечным пользователям.

## Ограничения по файлам

На момент подготовки пакета в Яндекс Вики один прикрепляемый файл не должен превышать 200 MB. Если архивы крупнее, лучше разбивать материалы на несколько вложений или хранить большие бинарники во внутреннем артефактном хранилище и давать ссылку из Вики.

## Практический совет по публикации

Для массового использования удобно разделить материалы на два слоя:

- публичный пользовательский слой — обзор, установка, использование, типовые ошибки;
- технический слой — архитектура, провайдеры, эксплуатация и поддержка.

Так конечным пользователям проще найти нужную информацию, а технической команде проще поддерживать проект отдельно.
